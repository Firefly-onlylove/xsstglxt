# 前端 UI 设计需求文档

> **使用说明：** 按照此文档逐页设计 UI，画好后把图片路径发给我。

---

## 设计规范

| 项目 | 要求 |
|------|------|
| 配色 | 你自己定（建议：主色蓝/绿，简洁清爽的校园风格） |
| 布局 | PC端为
主，响应式不做要求 |
| 页面宽度 | 建议 1200px - 1400px |
| 字体 | 中文，清晰易读 |

## 逐页详细设计说明

### P1 — 登录/注册页

**用途：** 所有用户的入口

**页面元素：**
- [ ] 标题文字：系统名称"学生社团管理系统" + 学校Logo（可选）
- [ ] 登录区域：
  - [ ] 用户名输入框
  - [ ] 密码输入框（掩码显示）
  - [ ] "登录"按钮
  - [ ] 登录失败时，红色文字提示错误信息
- [ ] "没有账号？立即注册"链接 → 切换到注册表单
- [ ] 注册区域（默认隐藏，点链接后切换显示）：
  - [ ] 真实姓名输入框
  - [ ] 用户名输入框
  - [ ] 密码输入框
  - [ ] 确认密码输入框
  - [ ] 学院下拉选择框（从数据库读取）
  - [ ] 专业下拉选择框（根据学院联动）
  - [ ] 班级下拉选择框（根据专业联动）
  - [ ] 学号输入框（10位）
  - [ ] 身份证号输入框（18位）
  - [ ] 手机号输入框（11位）
  - [ ] "注册"按钮
  - [ ] "已有账号？返回登录"链接

**操作流程：**
- 登录成功 → 根据角色跳转到对应主页
- 注册成功 → 提示"注册成功，请登录" → 切回登录

### P2 — 消息通知中心

**用途：** 所有角色查看收到的通知消息

**页面元素：**
- [ ] 顶部面包屑导航：首页 > 消息中心
- [ ] 左侧消息分类（未读/已读/全部）
- [ ] 右侧消息列表：
  - [ ] 每条消息显示：标题、内容摘要、时间、已读/未读标记
  - [ ] 点击展开查看完整内容
  - [ ] "全部标为已读"按钮
- [ ] 右上角未读消息数量角标（全局显示在导航栏）

**消息类型（不需要在UI显示，仅逻辑用）：**
社团申请结果 / 入社审批 / 活动取消 / 报销审核结果 / 投票通知 / 投票结果 / 被移出社团 / 职位变更 / 账号被限制 / 系统公告

### S1 — 学校管理员主页（仪表盘）

**用途：** 学校管理员登录后的首页，展示全校数据概览

**页面元素：**
- [ ] 顶部导航栏：
  - [ ] 左侧：系统名称
  - [ ] 中间：导航菜单（主页 / 社团管理 / 学院管理 / 用户管理 / 财务管理 / 系统设置）
  - [ ] 右侧：消息通知图标（带未读数量角标）+ 当前用户名 + 退出按钮
- [ ] 数据卡片区（一行4个卡片）：
  - [ ] 卡片1：活跃社团总数（数字大字 + "个社团"）
  - [ ] 卡片2：在籍成员总数
  - [ ] 卡片3：活动总场次
  - [ ] 卡片4：待审批社团数（红色高亮）
- [ ] 快捷操作区：
  - [ ] "社团审批"按钮 → 跳S2
  - [ ] "用户管理"按钮 → 跳S5
  - [ ] "添加学院"按钮 → 跳S4
  - [ ] "数据统计"按钮 → 下方统计区
- [ ] 下方统计区：
  - [ ] 社团分类饼图/柱状图（按类别统计）
  - [ ] 各学院社团数量对比（横向柱状图）
  - [ ] 最近操作日志列表（最近10条）
- [ ] 底部：系统公告栏 + 数据库备份按钮

### S2 — 社团审批页

**用途：** 审批待处理的社团创建申请

**页面元素：**
- [ ] 顶部导航栏（同S1，当前页高亮：社团管理）
- [ ] 左侧子导航：社团审批 / 全校社团列表
- [ ] 列表区：
  - [ ] 表格列：社团名称 / 类别 / 申请级别（校/院）/ 所属学院 / 创建者 / 申请时间 / 操作
  - [ ] 操作列按钮："通过"（绿色）、"驳回"（红色）
  - [ ] 顶部："待审批" / "已审批" 标签页切换
  - [ ] 每个标签页显示数量和分页
- [ ] 点击"通过" → 弹出模态框：
  - [ ] 确认通过？
  - [ ] 社团级别选择：校级 / 院级（下拉框）
  - [ ] 若选院级 → 选择所属学院（下拉框）
  - [ ] "确认通过"按钮 + "取消"按钮
- [ ] 点击"驳回" → 弹出模态框：
  - [ ] 驳回原因输入框（必填）
  - [ ] "确认驳回"按钮 + "取消"按钮
- [ ] 已审批标签页：表格多一列"审批结果"（通过/驳回）+ "驳回原因"

## 三、通用 UI 组件

以下组件在多个页面中复用，保持风格统一即可：

| 组件 | 出现位置 | 说明 |
|------|----------|------|
| 顶部导航栏 | 所有页面 | 角色不同菜单不同 |
| 数据卡片 | 仪表盘页面 | 标题+数字，可点击 |
| 数据表格 | 大部分列表页 | 排序+分页+行操作按钮 |
| 模态弹窗 | 审批/确认操作 | 居中弹出，背景半透明遮罩 |
| 筛选栏 | 列表页顶部 | 下拉框+搜索框+按钮组合 |
| 标签页切换 | 多视图页面 | 横向标签切换 |
| 分页控件 | 所有列表 | 上一页/下一页/页码 |
| 消息角标 | 导航栏右侧 | 红色圆形数字 |
| 面包屑导航 | 子页面顶部 | 层级路径 |
| 表单验证提示 | 所有输入页 | 红色文字，输入框下方 |
| 操作确认框 | 删除/解散等 | "确认操作？" + 确认/取消 |
| Toast提示 | 全局 | 操作成功/失败，右上角弹出，几秒后消失 |

## 四、页面跳转关系

```
                    ┌─── school_admin  → S1 学校主页
                    │
登录(P1) ──角色判断──┼─── college_admin → C1 学院主页
                    │
                    └─── student/general → T1 学生主页
```
