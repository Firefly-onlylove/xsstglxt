# 学生社团管理系统 PC端前端UI完整设计方案
## 一、基础设计规范（校园清爽蓝系）
### 1. 色彩体系
| 类型 | 色值 | 使用场景 |
| ---- | ---- | -------- |
| 主色 | #1677FF（浅天蓝） | 按钮、导航高亮、卡片标题、关键操作 |
| 辅助绿 | #36CFC9 | 成功、通过、报名、确认按钮 |
| 警示红 | #F53F3F | 驳回、禁用、待审批、超限、错误提示 |
| 中性灰1 | #F7F8FA | 页面背景、卡片底色 |
| 中性灰2 | #E5E6EB | 分割线、表格隔行 |
| 中性灰3 | #86909C | 次要文字、占位提示 |
| 文字黑 | #1D2129 | 正文、标题 |
| 遮罩 | rgba(0,0,0,0.45) | 弹窗底层蒙版 |

### 2. 布局&尺寸规范
1. 页面容器宽度：1300px，水平居中，左右留白自动适配
2. 字体：系统无衬线字体（Windows微软雅黑/Mac苹方）
   - 大标题：22px / 加粗
   - 模块标题：18px / 加粗
   - 正文文字：14px
   - 辅助小字：12px
3. 间距标准：
   - 模块外边距：24px
   - 卡片内边距：20px
   - 表单输入框间距：16px
   - 表格单元格内边距：12px
4. 圆角统一：卡片/输入框 8px，弹窗 12px，小按钮 6px

### 3. 全局通用组件UI规范
#### （1）顶部导航栏（多角色差异化）
高度60px，底色纯白，底部1px分割线#E5E6EB
- 左侧：系统Logo+文字「学生社团管理系统」
- 中间：角色专属导航菜单，hover/当前页文字主色+底部2px主色下划线
- 右侧：消息图标（红色圆形角标展示未读数量）+ 用户名下拉菜单 + 退出按钮

#### （2）数据统计卡片
宽高自适应，一行4等分，白底圆角8px，浅阴影；
内部：小字标签 + 超大加粗数字；右侧可配迷你折线图；底部可放快捷跳转按钮

#### （3）通用表格
- 表头：#F2F3F5 浅灰底色，文字加粗
- 隔行变色：偶数行 #F7F8FA
- 操作按钮组：内联小按钮，区分状态（绿色通过、红色驳回、蓝色编辑）
- 底部统一分页组件：页码、每页条数、跳转输入框

#### （4）弹窗Modal
居中固定弹窗，白底圆角12px；
顶部弹窗标题栏，底部操作区（取消/确认双按钮）；
背景半透明遮罩，点击遮罩可关闭弹窗；表单输入框自带红色校验提示文案

#### （5）筛选栏
横向布局：下拉选择器 + 搜索输入框 + 功能按钮，统一高度36px，组件间距12px

#### （6）标签页Tabs
横向平铺标签，未选中灰色文字，选中主色文字+底部主色下划线；切换无刷新渲染对应内容

#### （7）Toast全局提示
页面右上角悬浮弹窗，3s自动消失；成功绿色、失败红色、提示灰色

## 二、分页面UI详细设计（优先交付14页）
# P1 登录/注册页（独立落地页，无导航）
## 页面布局：左右分栏（左文案，右表单）
1. 左侧（占50%宽度，背景渐变#E8F3FF）
   - 顶部学校Logo
   - 大标题：学生社团管理系统
   - 副标题：校园社团一站式管理平台
   - 底部校园简约插画（社团活动矢量图）
2. 右侧（占50%，垂直居中表单卡片）
   - 白色圆角卡片，内边距40px
   - 默认展示【登录表单】
     - 用户名输入框、密码输入框（右侧眼睛显隐）
     - 红色文字错误提示（登录失败展示）
     - 蓝色「登录」主按钮（通栏）
     - 底部小字链接：没有账号？立即注册
   - 点击链接切换【注册表单】（隐藏登录）
     - 表单字段：姓名、用户名、密码、确认密码、学院下拉、专业联动下拉、班级联动下拉、10位学号、18位身份证、11位手机号
     - 实时输入校验，输入框下方红色提示
     - 蓝色「注册」通栏按钮
     - 底部返回登录链接
3. 交互：注册成功弹出绿色Toast，自动切回登录表单

# S1 学校管理员仪表盘主页
## 整体结构：顶部导航 → 数据卡片区 → 快捷操作区 → 图表统计区 → 底部公告栏
1. 顶部导航（学校管理员菜单：主页/社团管理/学院管理/用户管理/财务管理/系统设置）
2. 数据卡片一行4个：
   - 活跃社团总数卡片
   - 在籍成员总数卡片
   - 活动总场次卡片
   - 待审批社团（红色数字高亮）
3. 快捷操作按钮组：社团审批、用户管理、添加学院、数据统计
4. 图表区域（左右两栏）
   - 左：社团分类饼图
   - 右：各学院社团数量横向柱状图
5. 下方：最近10条操作日志表格
6. 底部：系统公告栏 + 数据库备份按钮

# S2 社团审批页
1. 顶部全局导航，当前「社团管理」高亮
2. 左侧二级子导航：社团审批 / 全校社团列表
3. 顶部标签切换：待审批（红色数字角标）、已审批
4. 列表表格字段：社团名称、类别、申请级别、所属学院、创建人、申请时间、操作
5. 操作按钮：绿色【通过】、红色【驳回】
   - 通过弹窗：选择社团级别（校级/院级）、选择学院、确认通过
   - 驳回弹窗：必填驳回原因输入框
6. 已审批标签表格增加两列：审批结果、驳回原因
7. 底部分页组件

# S5 用户管理页
1. 顶部筛选栏：角色下拉筛选 + 姓名/学号搜索框
2. 用户列表表格字段：ID、姓名、角色、学号、学院、手机号、账号状态、最后登录、操作
3. 操作按钮：查看详情、禁用/启用、重置密码、限制
4. 顶部功能按钮：蓝色「创建学院管理员」弹窗表单
5. 限制弹窗：限制类型（禁止入社/禁止活动）、起止时间、限制原因
6. 右侧悬浮面板：当前生效限制列表，支持解除限制
7. 分页控件

# C1 学院管理员仪表盘主页
1. 顶部导航菜单：主页/班级管理/社团监督/报销审批/学生管理
2. 4张数据卡片：本院社团数、社团总成员、活动总场次、待审批报销（红色）
3. 中部：本院社团简易列表，点击跳转社团详情
4. 底部：学院内部公告展示区

# C3 报销审批与额度管理页
双标签页：待审批、额度管理
## 标签1：待审批报销
1. 报销列表：社团名称、金额、报销说明、发票缩略图、申请人、提交时间、操作
2. 操作：查看发票大图弹窗、绿色通过、红色驳回（填写驳回理由）
## 标签2：额度管理
1. 本院社团单行配置项：社团名称、单次报销上限输入框、周期（月/学期）下拉、周期总额度、已使用额度、保存按钮

# T1 学生端社团广场主页
1. 顶部学生导航：社团广场/活动中心/我的社团/个人中心，右上角消息+用户名+退出，蓝色按钮「申请创建社团」
2. 搜索筛选区：
   - 顶部搜索框（检索社团名称）
   - 分类按钮组：全部/学术/文艺/体育/公益/实践
   - 级别筛选：全部/校级/院级
3. 社团卡片网格布局（一行4张卡片）
   - 卡片内容：社团Logo、名称、类别标签、级别标签、学院、成员数、简介
   - 卡片hover轻微上浮阴影，点击跳转T2社团详情
4. 底部分页

# T2 社团详情页
布局：左侧社团基础信息 + 右侧标签内容区
1. 面包屑导航：首页 > 社团详情 > XX社团
2. 左侧信息栏：
   - 大尺寸社团Logo
   - 社团名称、类别、级别、所属学院、指导老师、成员数量
   - 按钮区分身份：未加入显示【申请加入】；已加入普通学生显示【退出社团】
3. 右侧三标签页：
   - 成员列表：姓名、角色、入社时间
   - 活动列表：活动标题、时间、地点、查看详情跳转T4
   - 社团公告：公告标题+发布时间

# T4 活动详情页
1. 面包屑：首页 > 活动中心 > XX活动
2. 活动基础信息区：标题、主办社团、起止时间、地点、报名截止、已报名/人数上限、富文本活动描述
3. 状态按钮区分：
   - 未报名未满员：绿色【立即报名】
   - 已报名：灰色【取消报名】
   - 名额已满：红色标签「报名已满」
4. 活动进行中展示签到模块：6位签到码输入框+签到按钮，签到成功绿色提示
5. 活动结束后展示全部报名学生名单

# T6 我的社团页
双标签：已加入社团、申请记录
## 标签1 已加入社团
卡片列表，每张卡片展示Logo、社团名、成员数、活动数、自身角色；
社长/副社长卡片附带蓝色【进入管理后台】按钮，跳转M1；点击卡片进入T2详情
## 标签2 申请记录
列表：社团名称、申请时间、状态（待审/通过/驳回），驳回展示驳回原因

# M2 社团成员管理页
双标签：在籍成员、入社申请
## 在籍成员标签
表格：姓名、学号、社团角色、加入时间；
操作按钮：任命副社长/财务/秘书、移出社团；顶部姓名搜索框
## 入社申请标签
申请列表：姓名、学号、申请时间；绿色通过、红色拒绝按钮，操作后自动推送消息通知申请人

# M3 社团活动管理页
1. 顶部蓝色按钮【发布新活动】，弹出完整活动表单：标题、描述、地点、起止时间、报名截止、人数上限、签到开关
2. 活动列表表格：标题、时间、地点、报名人数、状态标签、操作按钮（随状态变化）
   - 草稿：编辑、发布、删除
   - 已发布：开始活动、取消活动（填写原因）
   - 进行中：结束活动、签到管理（跳转签到页面，展示6位签到码、签到名单）
   - 已完成：查看总结、归档
3. 取消活动弹窗填写原因，自动通知全部报名学生

# M4 社团财务管理页
1. 顶部财务数据卡片：总收入、总支出、当前结余、本月报销已用/总额度
2. 双标签：收支记录、报销申请
## 收支记录
筛选条件：收支类型、日期范围；表格展示金额、来源、说明、操作人、时间
## 报销申请
列表展示金额、说明、发票缩略图、审核状态、审核意见；
蓝色【新建报销】弹窗表单：金额、文字说明、发票图片上传预览、提交按钮；提交前校验报销额度上限

# M5 换届选举页
## 社长视角UI模块
1. 当前任期信息卡片：现任社长、副社长、任期时间
2. 状态提示横幅：是否存在进行中选举
3. 功能按钮：发起新一轮选举（弹窗配置副社长名额、报名/投票时间）、申请连任
4. 选举历史列表：每场选举状态、时间、报名人数、投票人数、查看结果按钮
## 普通社员视角UI
1. 顶部横幅提示选举状态
2. 报名阶段：候选人列表+【报名参选】（社长/副社长二选一）
3. 投票阶段：社长单选框、副社长多选框，提交投票（匿名仅一次投票）
4. 结果公示页面：得票排序展示当选人员，无投票人信息
5. 特殊状态展示：无人报名警示、平票重新投票、倒计时

## 三、页面跳转逻辑UI联动说明
1. 登录页P1根据账号角色跳转对应首页：
   - 学校管理员 → S1
   - 学院管理员 → C1
   - 普通学生 → T1
2. 学生T6我的社团，社长卡片点击【进入管理后台】跳转M1社团管理总后台，M1统一导航切换M2/M3/M4/M5
3. 所有页面右上角消息图标统一弹窗跳转公共页面P2消息通知中心
4. 所有列表弹窗操作（审批、解散、限制、取消活动）完成后自动刷新当前表格，弹出Toast提示操作结果

## 四、交付输出说明
如需要落地可视化效果图，可按以下格式输出：
1. 每张页面单独标注编号（如P1登录页、S1学校仪表盘）
2. 统一采用本套校园蓝色规范，组件复用统一风格
3. 区分角色页面视觉轻微差异化：
   - 学校管理员：主导航深色加重
   - 学院管理员：卡片增加浅绿边框区分
   - 学生端：卡片轻量化、圆角更大，风格年轻化
   - 社团社长后台：顶部导航增加「返回学生端」快捷按钮
