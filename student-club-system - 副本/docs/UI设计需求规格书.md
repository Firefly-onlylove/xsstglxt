# SCMS 学生社团管理系统 — UI 设计需求规格书

> **用途**：本文档供 UI 设计 AI 生成完整的页面设计稿（HTML/CSS 或 Figma 原型）。
> **原则**：本文档只描述**功能、数据、交互**，不预设视觉风格。具体配色、布局、组件样式由设计 AI 自由发挥。
> 目标用户：学校管理员、学院管理员、普通学生、社团管理员（社长/副社长）。

---

## 一、全局通用元素

### 1.1 顶栏（所有已登录页面）
- 左侧：系统 Logo/名称 "SCMS · 学生社团管理系统"
- 右侧：
  - 通知铃铛图标 + 未读数字角标（点击进入消息中心）
  - 用户头像/姓名（点击进入个人中心）
  - 退出登录按钮

### 1.2 侧边栏（可选，设计 AI 决定用顶栏导航还是侧边栏导航）
- 各角色的一级功能菜单

### 1.3 全局交互规范
- 删除/驳回等不可逆操作需弹窗二次确认
- 列表页均支持分页（页码 + 每页条数选择）
- 表格支持按列排序
- 加载状态显示骨架屏或 loading 动画
- 操作成功/失败显示 Toast 提示（顶部居中，3 秒自动消失）

## 二、登录页 `/login`

### 页面元素
| 元素 | 类型 | 说明 |
|------|------|------|
| 系统标题 | 文字 | "学生社团管理系统" + 英文副标题 SCMS |
| 用户名输入框 | input | placeholder: "用户名" |
| 密码输入框 | input[password] | placeholder: "密码"，支持回车提交 |
| 登录按钮 | button | 主色按钮，全宽 |
| 错误提示 | 文字 | 红色，显示在按钮下方 |
| 默认账号提示 | 文字 | 灰色小字："初始管理员 admin / admin123" |

### 交互流程
1. 输入用户名密码 → 点击登录或回车
2. 按钮变灰显示"登录中..."
3. 成功 → 跳转至角色对应首页
4. 失败 → 显示错误信息（用户名或密码错误 / 账号已被禁用）
