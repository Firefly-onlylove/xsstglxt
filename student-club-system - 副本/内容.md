# SCMS 学生社团管理系统 — 核心功能

## 项目定位

四角色社团全生命周期管理平台

## 角色与职责

| 角色 | 核心职责 |
|------|----------|
| 学校管理员 | 社团审批、社团管理、用户管理、财务监督、全校公告、数据统计 |
| 学院管理员 | 本院社团监督、学生管理（限制/解除）、报销审批、班级管理、学院公告 |
| 社团负责人 | 成员审批、活动发布管理、财务收支记录、报销申请、换届选举、社团公告 |
| 学生/社员 | 浏览社团广场、申请入社、报名活动、创建社团申请、消息中心 |

## 核心业务流程

1. 学生申请创建社团 → 学校审批通过 → 自动赋社长身份
2. 社员提交报销 → 学院审批（通过后自动记财务支出）或驳回
3. 社团发布活动 → 学生报名 → 签到统计
4. 多级消息广播（学校→全员、学院→本院及以下、社团→本社成员）

## 技术栈

- 后端：C11 + Mongoose 嵌入式 HTTP 库 + MySQL 8.0 C API
- 前端：Vue 3.4 + Vite 5.2 + Element Plus 2.7 + Vue Router 4.3
- 数据库：MySQL 8.0（21张表，5个触发器）
