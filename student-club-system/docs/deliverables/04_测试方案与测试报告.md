### 四、缺陷发现与修复记录（关键缺陷）

| 编号 | 严重级别 | 缺陷描述 | 根因 | 修复方案 |
|------|---------|---------|------|---------|
| BUG-01 | P0 | 社长登录后社团管理模块不显示（role 为 general_student） | 社长/副社长的 users.role 未同步为 club_admin | dynamic role check in /api/me（检查 members 表） |
| BUG-02 | P0 | "设为管理员"报"成员不存在或为社长" | 前端传 user_id，后端取 member_id，参数不匹配 | 前端改用 row.member_id 调 API |
| BUG-03 | P1 | 副社长可任命管理员 | 权限校验用 club_require_manager，应仅社长可操作 | 改用 club_require_president |
| BUG-04 | P1 | 报销提交失败（status='college_pending' 非有效 ENUM 值） | 后端写入了未定义的 ENUM 值 | 改为合法的 status='pending' |
| BUG-05 | P1 | 财务记录页面 operator_name 不显示 | SQL 查询用了不存在的列名 | LEFT JOIN users 取 real_name |
| BUG-06 | P2 | 报销列表 purpose 列不显示 | 前端 prop 为 'purpose'，后端返回 'description' | 前端 prop 改为 'description' |
| BUG-07 | P2 | 审批操作失败无错误提示 | 缺少 else 错误分支 | 补全 ElMessage.error(res.msg) |
| BUG-08 | P2 | 仪表盘待处理报销数不准确 | 统计时未排除学院已批的报销 | college_reviewed='pending' 过滤 |
| BUG-09 | P3 | 成员退出社团后仍有未读消息 | member_id 误当 user_id 传递给API | 统一使用 member_id |
