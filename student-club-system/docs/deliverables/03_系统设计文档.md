## 一、整体架构设计

### 1.1 架构概述

SCMS 采用**单体 Web 应用架构**，后端使用 C11 语言编译为单一可执行文件，内嵌 Mongoose HTTP 库提供服务，前端为 Vue 3 SPA，数据持久化使用 MySQL 8.0。

### 1.2 技术选型

| 决策点    | 选择                         | 理由                                                                          |
| --------- | ---------------------------- | ----------------------------------------------------------------------------- |
| 后端语言  | C11 (GCC/MinGW-w64)          | 实训核心要求，全面运用 C 语言知识点（指针、结构体、文件操作、动态内存、算法） |
| HTTP 框架 | Mongoose (单文件嵌入)        | 轻量级 HTTP 库，单文件 .c/.h 即可集成，零外部依赖                             |
| 数据库    | MySQL 8.0 + C API            | 成熟的关系型数据库，C API 直接控制连接和查询                                  |
| 前端框架  | Vue 3 + Vite 5               | 组件化开发、响应式数据绑定、按需编译                                          |
| UI 组件库 | Element Plus 2.7             | 企业级组件，表格/表单/弹窗/标签开箱即用                                       |
| 前端路由  | Vue Router 4 (Hash 模式)     | 纯前端路由，不依赖服务端 History 配置                                         |
| 认证方案  | SHA256 + Cookie + 内存会话池 | 自行实现密码哈希和 Token 管理，不依赖 OpenSSL                                 |
| 构建工具  | Makefile (mingw32-make)      | 简洁构建，区分 debug/release                                                  |

### 2.2 模块职责与 C 语言实现要点

#### 2.2.1 认证模块 (auth.c)

- **密码哈希**：自行实现 SHA256，通过宏展开位操作（右移、异或、与、Sigma 函数），无外部加密库依赖
- **Token 生成**：使用 `rand()` 配合时间种子生成 32 位十六进制随机字符串
- **会话池**：静态数组实现的哈希表（1024 槽位），开放寻址法处理冲突。每次请求通过 token 查找用户信息（包括重新查询数据库确保角色实时性）
- **Cookie 下发**：通过 Mongoose 的 `mg_http_reply` 在响应头中 `Set-Cookie`

#### 2.2.2 数据库连接池 (db.c)

- **连接管理**：`MYSQL*` 指针数组 + `pthread_mutex_t` 互斥锁保护
- **获取/归还**：`db_get_conn()` 轮询查找空闲连接并上锁，`db_release_conn()` 解锁归还
- **参数化查询**：`db_query(fmt, ...)` 和 `db_execute(fmt, ...)` 封装 `mysql_real_query`，字符串参数用 `'%s'` 格式化（已转义）、数值用 `%d`
- **事务接口**：`db_begin()`/`db_commit()`/`db_rollback()` 封装 `mysql_autocommit` + `mysql_commit`/`mysql_rollback`
- **便捷取值**：`db_query_int()`/`db_query_double()`/`db_query_str()` 直接返回单值

#### 2.2.3 API 辅助层 (api.c)

- **参数读取与 SQL 注入防护**：
  - `api_get_param()` 通过 Mongoose 的 `mg_http_get_var` 直接读取，不对值做额外处理
  - 在 `db_query` 的格式化参数中，**字符串值自动经过 `db_escape()` 转义**（内部调用 `mysql_real_escape_string`），然后再拼入 SQL
  - **数值参数使用 `%d` 格式化**，编译器保证类型安全，无法注入
  - 动态分配的内存在转义后立即 `free()`，避免内存泄漏
- **JSON 响应**：`api_ok()`、`api_ok_data()`、`api_error()` 系列函数通过 `mg_http_reply` 统一返回 `{"code":<code>,"msg":"<msg>","data":<data>}` 格式，HTTP 状态码统一 200
- **权限层**：`api_require_role()`、`api_require_school_admin()`、`api_require_college_admin()`、`api_require_club_admin()` 返回 0/1 供调用方判断

#### 2.2.4 路由系统 (routes.c)

- **路由表**：静态 `Route` 结构体数组，每项包含 `method`（"GET"/"POST"/"PUT"/"DELETE"）、`pattern`（URI 模式，`*` 匹配单一路径段）、`handler`（函数指针）、`need_login` 标志
- **模式匹配**：`uri_match()` 在 main.c 中实现，逐段对比（`*` 通配跳过任意一段），不跨 `/` 边界
- **排序要求**：更具体的路由（如 `/api/school/clubs/pending`）必须在通配路由（`/api/school/clubs/*`）之前

### 3.3 触发器

| 触发器                       | 作用                                | 触发时机                                     |
| ---------------------------- | ----------------------------------- | -------------------------------------------- |
| trg_member_count_insert      | 成员加入时 club.member_count +1     | members INSERT (join_status='approved')      |
| trg_member_count_update      | 成员退出时 club.member_count -1     | members UPDATE (left_at 从 NULL 变为非 NULL) |
| trg_member_count_delete      | 成员记录删除时 club.member_count -1 | members DELETE                               |
| trg_participant_count_insert | 活动报名时 participant_count +1     | activity_registrations INSERT                |
| trg_participant_count_delete | 取消报名时 participant_count -1     | activity_registrations DELETE                |

## 六、安全设计

### 6.1 认证安全

- 密码使用 SHA256 哈希存储，不明文保存（自行实现，200+ 行纯 C 算法）
- 登录后下发 32 位十六进制随机 Token，暴力枚举空间为 16^32
- Cookie 设置 HttpOnly 标志，JavaScript 无法读取

### 6.2 SQL 注入防护

- 所有字符串参数在拼入 SQL 前通过 `db_escape()` 转义（内部调用 `mysql_real_escape_string`）
- 数值参数使用 `%d` / `%.2f` 格式化，类型安全
- 没有使用字符串拼接构造 SQL，全部通过格式化参数

### 6.3 文件上传安全

- 文件名随机化：时间戳 + 6 位随机码，避免路径穿越和覆盖
- 扩展名校验：只保留字母和数字字符，过滤特殊字符
- 限制上传类型：仅接受图片格式

### 6.4 权限安全

- 前端路由守卫校验角色
- 后端每个 handler 入口调用 `api_require_*` 系列函数
- 数据归属校验：社团操作需验证 operator 属于该社团
