# SCMS 答辩 PPT 生成模板

## 使用说明

> 以下字段请用实际值替换模板中的占位符。每个幻灯片包含 **标题**、**内容要点** 和 **配图建议** 三部分。
> 将本文档发给 AI（如 Gamma.app、美图 AI PPT、讯飞智文、iSlide AI 等），配合"生成答辩 PPT"指令即可。

### 第 8 页：数据库设计

- **标题**：数据库设计 (ER 模型)
- **内容**：
  - 21 张数据表，覆盖学院/专业/班级/用户/社团/成员/活动/财务/报销/选举/通知/日志
  - 5 个触发器：成员计数自动维护、活动报名计数、财务报销联动记账
  - InnoDB 引擎，utf8mb4 字符集
- **核心关系**：
  - colleges → majors → classes (三级组织)
  - clubs ←→ members ←→ users (社团成员三角)
  - clubs → activities → registrations (活动报名链)
- **配图建议**：简化版 ER 图，突出核心 6 张表的关系

---

### 第 9 页：API 接口设计

- **标题**：RESTful API 设计
- **内容**：
  - 约 100 个 API 端点，统一 JSON 响应格式 `{"code":0,"msg":"ok","data":{...}}`
  - 认证方式：Cookie `scms_token` + SHA256 密码哈希
  - 权限控制：路由级 need_login + 业务级 api_require_role + 数据行级校验
  - URI 通配匹配 `*`，支持路径参数提取
- **关键端点展示**：

| 方法 | 端点 | 作用 |
|------|------|------|
| POST | /api/login | 用户登录 |
| GET | /api/school/dashboard | 学校仪表盘 |
| POST | /api/school/clubs/{id}/approve | 审批建社 |
| GET | /api/my/clubs | 我的社团 |
| POST | /api/club/{id}/reimbursements | 提交报销 |

- **配图建议**：API 请求/响应流程图

### 第 10 页：安全设计

- **标题**：安全设计
- **内容** (4 个安全维度)：
  - **密码安全**：SHA256 哈希存储 (64 位十六进制)，不可逆
  - **会话安全**：32 位随机 Token，HttpOnly Cookie，防 XSS 窃取
  - **SQL 注入防护**：`mysql_real_escape_string` 转义所有用户输入，数值参数类型安全
  - **文件上传安全**：随机文件名 (时间戳+随机码)，扩展名白名单，目录穿越防护
- **配图建议**：四宫格安全图标 (锁/盾牌/防火墙/文件夹)
