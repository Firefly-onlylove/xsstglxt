# SCMS 学生社团管理系统 — 两设备 GitHub 协作开发方案

## 一、当前痛点

| 问题 | 原因 |
|------|------|
| `public/assets/` 每次构建都冲突 | `.gitignore` 未忽略构建产物 |
| `db_config.ini` 两台设备配置不同 | 数据库密码不一样，互相覆盖 |
| 数据库变更难同步 | SQL 脚本在仓库，需手动执行 |
| 同时改一个文件产生冲突 | 无明确的分工划分 |

## 二、Git 配置修改

### 2.1 忽略构建产物

在 `student-club-system/.gitignore` 追加：

```
# 前端构建产物（每台设备各自 npm run build）
public/
```

已入库的 `public/` 需要执行一次移除：

```bash
git rm -r --cached student-club-system/public/
git commit -m "chore: 移除 public/ 构建产物，改用 gitignore 忽略"
```

### 2.2 数据库配置模板化

1. 将 `db_config.ini` 复制为 `db_config.example.ini`（模板文件，跟踪）
2. 在 `.gitignore` 中追加 `db_config.ini`（真实配置，不跟踪）

```bash
cp student-club-system/db_config.ini student-club-system/db_config.example.ini
echo "db_config.ini" >> student-club-system/.gitignore
git rm --cached student-club-system/db_config.ini
```

3. 每台设备首次使用时，复制模板并填入自己的密码：

```bash
cp db_config.example.ini db_config.ini
# 然后编辑 db_config.ini 修改 user 和 password
```

## 三、分支策略

```
main (1)       ← 稳定版本，只合并不直接修改
 ├── dev       ← 日常开发分支，两人在此合并
      ├── fix/xxx    ← 修复 bug
      └── feat/xxx   ← 新功能
```

### 日常开发流程

```bash
# 早上开始工作
git checkout dev
git pull origin dev

# 开发新功能
git checkout -b feat/your-feature-name
# ... 写代码 ...
git add <只加源文件，不加 public/>
git commit -m "feat: 功能描述"
git push origin feat/your-feature-name

# 合并到 dev
git checkout dev
git merge feat/your-feature-name
git push origin dev
```

### 提交规范

| 前缀 | 用途 | 示例 |
|------|------|------|
| `fix:` | 修 bug | `fix: 修复禁用用户无反馈` |
| `feat:` | 新功能 | `feat: 添加社团活动签到` |
| `chore:` | 杂务/配置 | `chore: 更新 gitignore` |
| `docs:` | 文档 | `docs: 添加协作开发方案` |

## 四、数据库同步

两台设备各自连接本地 MySQL，数据互不影响。

### 结构变更同步

谁改了表结构，谁改 `database/02_create_tables.sql`。另一方拉取后：

```bash
# 只重建表结构（不影响已有数据，需手动对比执行）
# 或完整重置：
cmd /c '"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p --default-character-set=utf8mb4 -e "DROP DATABASE IF EXISTS scms_db"'
# 然后依次导入 01~05 脚本
```

### 测试数据

统一使用 `database/05_mock_data.sql` 维护测试数据。任何一方修改了 mock 数据需告知另一方重建数据库。

## 五、分工建议

按模块划分，减少同时编辑同一文件：

| 模块 | 负责人 | 目录 |
|------|--------|------|
| 学校端 API | A | `server/src/school/` |
| 学院端 API | A/B | `server/src/college/` |
| 学生端 API | B | `server/src/student/` |
| 公共模块 | A | `server/src/common/` |
| 前端页面 | B | `client/src/views/` |
| SQL 建表 | A | `database/02_*.sql` |
| 测试数据 | B | `database/04_*.sql`, `05_*.sql` |

## 六、新设备上手步骤

```bash
# 1. 克隆项目
git clone https://github.com/Firefly-onlylove/7.git
cd 7
git checkout dev

# 2. 配置数据库连接
cd student-club-system
cp db_config.example.ini db_config.ini
# 编辑 db_config.ini，修改 MySQL root 密码

# 3. 初始化数据库
# 用 MySQL root 账号依次导入 database/01~05

# 4. 编译后端
cd server
mingw32-make clean && mingw32-make

# 5. 编译前端
cd ../client
npm install && npm run build

# 6. 启动
cd ..
scms.exe
# 访问 http://127.0.0.1:8000
```
